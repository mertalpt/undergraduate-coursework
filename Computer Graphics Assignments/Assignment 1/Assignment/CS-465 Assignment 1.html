Requirements

You will design and implement a simple procedural WebGL app that will paint a very simple cityscape. You need to generate the paintings using these simple parameters:

  • number of houses (n): In the painting, there will be n houses exactly.
  • max number of stories (m): Each house can have stories up to m (determine the number of stories for each house between 1 and m randomly) and will have random colors except the roof and the windows.
Between each pair of houses, there will be a tree. There should be a ground plane, sky and the sun. Clouds will be drawn interactively. Once you generate the cityscape using the parameters, you should be able to click on the canvas to add corner vertices of a cloud. Once you hit right-click (or some other key), this will indicate that cloud drawing is finished. Users can paint as many clouds as they want. In the UI, there should be a slider for each parameter (number of houses and max number of stories). There should be a button named "Generate". When pressed, a new cityscape should be generated from scratch. In the following figure sample cityscape drawing is shown (number of houses = 4, max number of stories = 3).

You need to use suitable GL primitives for each entity (quads, tris etc). Don't hardcode the coordinates of the vertices. For example, circular entities (sun and tree tops) should have at least 32 segments which you can compute using trigonometric functions. You should be careful about the order of drawings (clouds should be in front of the sun etc.). Don't be afraid to get creative! You could add random variations to entities to generate better looking drawings. Also, additional effects & entities are welcome! Since each drawing will be random, you might want to save them if one of them looks interesting. Provide a save & load mechanism which saves the properties of the drawings (coordinates and colors of the houses, windows etc.). You should be able to load these files from disk and get the exact drawing. (Don't save the whole framebuffer pixel by pixel!). You can also save the whole drawing by just saving the random seed and the two parameters. However, for this assignment, you are not allowed to do that.

Important: Always comment your code. The code will also be checked during the demos.
